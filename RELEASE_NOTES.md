# 发布说明

## [v0.3.0-alpha] - 2026-03-20

### 🚀 阶段切换
- 正式将开发主线从 `v0.2.x-alpha` 切换到 `v0.3.0-alpha`
- 项目进入玩法主循环重构阶段（Monster / Turn / Level / Event / Reward / Save）

### 🧠 玩法主线调整（新增）
- 明确将“地雷”重定义为“怪物显形点”，揭雷不再直接判负
- 保留扫雷信息层（邻格数字推理），新增卡牌战术层（资源与战斗处理）
- `Phase 3` 优先目标调整为“卡牌主导的最小 Run 闭环”
- 后续平衡基线从“是否可扫雷通关”改为“是否必须合理出牌才能稳定通关”

### ✅ 来自 Phase 2 的收尾成果
- 统一卡牌交互提示文案（目标选择 / 二次确认 / 能量不足）
- 目标可用性反馈增强（可选/不可选目标预览）
- 取消交互统一（ESC / 点击空白区域 / 取消按钮）
- 调试日志收口并统一到 `CONFIG.debug.enabled`
- 新增 `SMOKE_TEST.md`，沉淀关键流程冒烟回归清单
- 核心文档补齐（状态流、事件命名、模块说明）

### 📌 当前已知边界
- 移动端触控体验已可用，但“深度打磨”作为后续 UX 专项继续推进
- 仍处于 Alpha 阶段，P3-A 已落地，后续聚焦 P3-B（关卡/事件/奖励/存档）

### 🎯 下一步（v0.3.0-alpha）
- 关卡推进 + 事件 + 奖励三选一基础流程
- Run 状态与本地存档基础骨架
- 怪物类型与卡池扩展（平衡回归基线）

### 🧪 v0.3.0-alpha 增量实现（开发中）
- 新增最小怪物遭遇闭环：揭雷后进入“显形怪物待处理”而非立即判负
- 新增遭遇处理方式：可用 `scout` 定点处理显形怪物，或硬闯承受 1 点生命损失
- 新增遭遇约束：存在显形怪物时，需先完成处理后再继续常规揭示
- 新增最小战斗结算：显形怪物具备 HP/反击，卡牌可造成伤害并触发结算
- 新增回合循环：遭遇结算后自动回合推进，重置能量并补充手牌
- 新增胜利约束：仅清空安全格不足以获胜，需完成全部显形怪物处理

## [v0.1.0-alpha] - 2026-03-13

### 🎮 功能特性
- **核心玩法**: 完整的扫雷实现
  - 左键点击揭开方块
  - 右键点击标记地雷
  - 首次点击保护（第一次点击后生成地雷）
  - 自动揭开空白方块（泛洪填充）
  - 胜利/失败判定

- **渲染**: 基于Canvas的网格渲染
  - 彩色数字（1-8）
  - 不同方块状态的视觉反馈
  - 流畅的60 FPS渲染

- **用户界面**:
  - 主菜单
  - 抬头显示器（地雷计数器、计时器）
  - 胜利/游戏结束对话框
  - 简洁的深色主题界面

### 🏗️ 技术架构
- 使用现代JavaScript的ES6模块
- 事件驱动架构（EventBus）
- 状态管理系统
- 基于requestAnimationFrame的游戏循环
- 模块化、可维护的代码结构

### 🐛 已知问题
- 尚未实现卡牌系统（计划于v0.2.0）
- 尚未实现Roguelike元素（计划于v0.3.0）
- 基础视觉效果，无美术资源
- 无音效
- 未针对移动端触摸操作优化

### 🔧 浏览器支持
- Chrome/Edge: ✅ 完整测试
- Firefox: ✅ 完整测试
- Safari: ⚠️ 基础测试
- 移动浏览器: ⚠️ 有限支持

### 📊 性能指标
- 首次加载: ~1.5秒
- 运行时帧率: 60 FPS
- 打包大小: ~15KB（未压缩）

### 🎯 下一版本 (v0.2.0-alpha)
- 卡牌系统实现
- 卡组构建机制
- 基础卡牌效果
- 能量系统
