# 发布说明

## [v0.3.0-alpha] - 2026-03-20

### 🚀 阶段切换
- 正式将开发主线从 `v0.2.x-alpha` 切换到 `v0.3.0-alpha`
- 项目进入玩法主循环重构阶段（Monster / Turn / Level / Event / Reward / Save）

### 🧠 玩法主线调整（新增）
- 明确将“地雷”重定义为“怪物显形点”，揭雷不再直接判负
- 保留扫雷信息层（邻格数字推理），新增卡牌战术层（资源与战斗处理）
- `Phase 3` 优先目标调整为“卡牌主导的最小 Run 闭环”
- 后续平衡基线从“是否可扫雷通关”改为“是否必须合理出牌才能稳定通关”

### ✅ 来自 Phase 2 的收尾成果
- 统一卡牌交互提示文案（目标选择 / 二次确认 / 能量不足）
- 目标可用性反馈增强（可选/不可选目标预览）
- 取消交互统一（ESC / 点击空白区域 / 取消按钮）
- 调试日志收口并统一到 `CONFIG.debug.enabled`
- 新增 `SMOKE_TEST.md`，沉淀关键流程冒烟回归清单
- 核心文档补齐（状态流、事件命名、模块说明）

### 📌 当前已知边界
- 移动端触控体验已可用，但“深度打磨”作为后续 UX 专项继续推进
- 仍处于 Alpha 阶段，P3-A 已落地，后续聚焦 P3-B（关卡/事件/奖励/存档）

### 🎯 下一步（v0.3.0-alpha）
- 关卡推进 + 事件 + 奖励三选一基础流程
- Run 状态与本地存档基础骨架
- 怪物类型与卡池扩展（平衡回归基线）

### 🧪 v0.3.0-alpha 增量实现（开发中）
- 新增最小怪物遭遇闭环：揭雷后进入“显形怪物待处理”而非立即判负
- 新增遭遇处理方式：可用战斗卡牌处理显形怪物，或硬闯承受生命损失
- 新增遭遇约束：存在当前显形怪物时，常规揭示受阻，需先处理怪物
- 新增最小战斗结算：显形怪物具备 HP/反击，卡牌可造成伤害并触发结算
- 新增回合循环：遭遇结算后自动回合推进，重置能量并补充手牌
- 新增胜利约束：仅清空安全格不足以获胜，需完成全部显形怪物处理
- 新增数据驱动怪物系统：`monsterDefinitions` 支持多怪物类型、权重与成长模板
- 修复怪物显形可视化时机：踩雷触发遭遇后，怪物会在触发格即时显示
- 新增怪物意图模型：遭遇显示 `intent`（下一步行为），敌方结算按意图伤害执行
- 新增硬闯递增惩罚：连续硬闯会提高承伤，避免成为长期最优策略
- 新增战斗设计文档：`COMBAT_DESIGN.md`（v0.1 初稿，用于 P3-B 实现对齐）
- 新增怪物悬浮信息：鼠标停在显形怪物格上可持续查看名称/状态/HP/意图/攻击
- 新增幕结构设计文档：`ACT_STRUCTURE_DESIGN.md`（Act-as-Grid v0.1 初稿）
- 新增卡牌职责矩阵文档：`CARD_ROLE_MATRIX.md`（战斗击杀手段与怪物克制初稿）
- 新增首轮战斗卡池：`strike`、`chain_probe`、`armor_break`、`guard`、`smoke_screen`
- 新增战斗标签机制：卡牌支持 `baseDamage/attackTag/combatOnly/tags`
- 新增怪物克制字段：`resistanceTag/tagModifiers`，按伤害标签修正结算
- 新增战斗状态效果：`易伤(vulnerableTurns)` 与 `烟幕覆盖(smokeScreenTurns)`
- 调整起始牌组到“可稳定击杀”结构，降低“只能靠侦察卡输出”的问题
- 新增能量与抽牌循环文档：`ENERGY_DRAW_LOOP_DESIGN.md`（翻格供能 + 主动抽牌 + 回洗规则）
- 调整供能规则口径：从“每个安全格 +1”改为“每次翻格操作 +1（单次操作上限 +1）”
- 实装探索供能：每次翻格操作可获得能量（单次操作上限 +1，且受最大能量约束）
- 实装遭遇保底能量：进入显形怪物遭遇时，最低补至 2 点
- 实装主动动作“抽牌”：消耗 1 能量，抽 1 张牌（每回合 1 次）
- 接入每回合能量恢复与抽牌动作次数重置，补全战斗经济循环
- 调整卡池与清算逻辑：`scout` 已从当前卡池移除，避免绕过遭遇处理
- 调整卡池复杂度：`mine_detector` 已从当前卡池移除，侦测类机制延后到 P3-C
- 起始牌组配置外置：改为 `CONFIG.player.startingDeck` 驱动
- 新增牌库查看入口：手牌控制区增加“牌库”按钮与总携带卡数量显示
- 升级牌堆查看界面：牌库/弃牌堆改为全屏覆盖层，查看时隔离游戏主区域交互
- 统一卡牌展示样式：牌库/弃牌堆界面改为与手牌区一致的卡牌视觉结构（能量/类型/名称/描述）
- 调整怪物点击交互：战斗中再次点击已显形怪物只展示信息，不再触发“自伤硬闯”
- 调整遭遇探索规则：恢复“当前显形怪物会阻挡探索”的主规则
- 调整烟幕卡机制：`smoke_screen` 改为覆盖目标 2 回合，期间该怪物不阻挡探索

### ✅ 本轮验收（战斗开始部分）
- 验收结论：阶段性通过
- 通过点：
  - 显形怪物已具备可被稳定击败的首轮卡牌手段（输出/防御/控制/资源）
  - 怪物意图与标签克制进入实战结算，战斗决策层成立
  - 起始牌组可支撑首局基础战斗闭环
- 未纳入本轮范围：
  - 关卡推进、事件、奖励三选一、Run 存档（P3-B 后续）

## [v0.1.0-alpha] - 2026-03-13

### 🎮 功能特性
- **核心玩法**: 完整的扫雷实现
  - 左键点击揭开方块
  - 右键点击标记地雷
  - 首次点击保护（第一次点击后生成地雷）
  - 自动揭开空白方块（泛洪填充）
  - 胜利/失败判定

- **渲染**: 基于Canvas的网格渲染
  - 彩色数字（1-8）
  - 不同方块状态的视觉反馈
  - 流畅的60 FPS渲染

- **用户界面**:
  - 主菜单
  - 抬头显示器（地雷计数器、计时器）
  - 胜利/游戏结束对话框
  - 简洁的深色主题界面

### 🏗️ 技术架构
- 使用现代JavaScript的ES6模块
- 事件驱动架构（EventBus）
- 状态管理系统
- 基于requestAnimationFrame的游戏循环
- 模块化、可维护的代码结构

### 🐛 已知问题
- 尚未实现卡牌系统（计划于v0.2.0）
- 尚未实现Roguelike元素（计划于v0.3.0）
- 基础视觉效果，无美术资源
- 无音效
- 未针对移动端触摸操作优化

### 🔧 浏览器支持
- Chrome/Edge: ✅ 完整测试
- Firefox: ✅ 完整测试
- Safari: ⚠️ 基础测试
- 移动浏览器: ⚠️ 有限支持

### 📊 性能指标
- 首次加载: ~1.5秒
- 运行时帧率: 60 FPS
- 打包大小: ~15KB（未压缩）
