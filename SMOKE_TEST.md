# 冒烟测试清单

适用版本：`v0.3.x-alpha`（怪物化扫雷 + Run 核心阶段）

## 执行方式

- 本地启动：`python -m http.server 8080`
- 访问：`http://localhost:8080`
- 建议每次提交前至少完整跑 1 次；发布前跑 3 次。

## 核心流程（必须通过）

- [ ] 1. 菜单进入对局：点击 `Start Game` 后可正常开局，无报错。
- [ ] 2. 基础网格：左键揭示、右键标记、数字显示正常。
- [ ] 3. 关键主线：揭示地雷后进入怪物处理流程（不应立即 Game Over）。
- [ ] 4. 战斗结算：显形怪物存在 HP/反击，使用战斗输出卡可生效并正确结算。
- [ ] 5. 失败流程：资源耗尽或生命归零时可正常触发 `Game Over`。
- [ ] 6. 手牌渲染：开局有初始手牌，能量与手牌可用态同步更新。
- [ ] 7. 无目标卡二次确认：首次点击进入确认态，二次点击同卡后生效并扣能量。
- [ ] 8. 需目标卡选择：点击卡牌后出现“请选择目标格子”提示，点击网格后生效。
- [ ] 9. 能量不足反馈：能量不足时给出明确提示（包含需要值和当前值），且不会误消耗卡牌。
- [ ] 10. 侦测卡状态：`mine_detector` 在当前版本已移除，不应进入首轮战斗卡池。
- [ ] 11. 主动抽牌动作：点击“抽牌（1⚡）”后消耗 1 点能量并抽 1 张牌，且每回合最多使用 1 次。
- [ ] 12. 版本角标：右上角 `git` 短哈希可显示（未知环境下允许 `unknown`）。
- [ ] 13. 回合循环：每次遭遇结算后进入新回合，能量重置且手牌按规则补充。
- [ ] 14. 胜利约束：仅清空安全格不应直接胜利，需处理全部怪物后才触发 Victory。
- [ ] 15. 遭遇阻挡约束：存在当前显形怪物时，非目标格应被阻挡；点击怪物格仅显示信息，不应触发承伤。
- [ ] 16. 烟幕卡放行：对当前怪物使用 `smoke_screen` 后，2 回合内应允许继续揭示其他区域；到期后恢复阻挡。
- [ ] 17. 事件未知性：玩家翻开事件格前不应知晓具体事件类型；翻开后才显示敌人/NPC/奖励类型。
- [ ] 18. Boss 解锁门槛：发现 Boss 节点后，若战斗事件完成数不足或未完成困难/精英战斗，不应允许挑战 Boss。
- [ ] 19. 敌人分层收益：普通/困难/精英战斗事件结算金币应符合递增关系（普通 < 困难 < 精英）。
- [ ] 20. 商店事件：商店应提供多档位商品、刷新按钮，且同一幕内可重复访问并继续购买。
- [ ] 21. 事件状态流：同一事件节点应遵循 `hidden -> revealed -> pending -> resolved` 顺序推进，不应发生回退。
- [ ] 22. 主循环连续性：可稳定复现“翻格 -> 事件触发 -> 处理结算 -> 继续翻格”且无卡死。

## 交互回归（建议通过）

- [ ] 23. ESC 取消目标选择：进入目标选择后按 ESC 可退出，不残留卡牌选中态。
- [ ] 24. 切换选卡稳定性：目标选择中点击另一张需目标卡，可平滑切换，不堆叠异常状态。
- [ ] 25. 移动端触控：短按揭示、长按插旗可用，无明显误触回归。
- [ ] 26. 取消交互：目标态/确认态下可通过“点击空白区域”或“取消按钮”退出，不消耗能量。
- [ ] 27. 牌堆界面层级：打开牌库/弃牌堆后应全屏覆盖游戏区，菜单/网格不应抢焦点。
- [ ] 28. 牌堆界面样式：牌库/弃牌堆中的卡牌视觉信息应与手牌区一致（能量/类型/名称/描述）。

## 判定标准

- 全部“核心流程”通过，且无明显控制台异常。
- 若“交互回归”失败，需在 `RELEASE_NOTES.md` 标记已知问题后再决定是否发布。

## 说明

- 当前代码仍在从 `Phase 2` 玩法迁移到怪物化扫雷主线的过程中。
- 若某条目尚未实装，需在提测记录中明确标记“未实现（非回归失败）”。
