# Phase 2 冒烟测试清单

适用版本：`v0.2.x-alpha`（卡牌系统阶段）

## 执行方式

- 本地启动：`python -m http.server 8080`
- 访问：`http://localhost:8080`
- 建议每次提交前至少完整跑 1 次；发布前跑 3 次。

## 核心流程（必须通过）

- [ ] 1. 菜单进入对局：点击 `Start Game` 后可正常开局，无报错。
- [ ] 2. 基础扫雷：左键揭示、右键插旗、数字显示正常。
- [ ] 3. 胜负流程：能正常触发 `Victory` / `Game Over` 对话框，按钮可继续游戏或回菜单。
- [ ] 4. 手牌渲染：开局有初始手牌，能量与手牌可用态同步更新。
- [ ] 5. 无目标卡二次确认：首次点击进入确认态，二次点击同卡后生效并扣能量。
- [ ] 6. 需目标卡选择：点击卡牌后出现“请选择目标格子”提示，点击网格后生效。
- [ ] 7. 能量不足反馈：能量不足时给出明确提示（包含需要值和当前值），且不会误消耗卡牌。
- [ ] 8. `mine_detector` 高亮：探测后有高亮反馈，延时后自动清理高亮。
- [ ] 9. `extra_life` 生效：踩雷时若有额外生命会消耗并继续对局。
- [ ] 10. 版本角标：右上角 `git` 短哈希可显示（未知环境下允许 `unknown`）。

## 交互回归（建议通过）

- [ ] 11. ESC 取消目标选择：进入目标选择后按 ESC 可退出，不残留卡牌选中态。
- [ ] 12. 切换选卡稳定性：目标选择中点击另一张需目标卡，可平滑切换，不堆叠异常状态。
- [ ] 13. 移动端触控：短按揭示、长按插旗可用，无明显误触回归。

## 判定标准

- 全部“核心流程”通过，且无明显控制台异常。
- 若“交互回归”失败，需在 `RELEASE_NOTES.md` 标记已知问题后再决定是否发布。
