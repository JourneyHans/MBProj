# ui 模块开发文档（更新）

`js/ui/` 用于沉淀可复用 UI 组件逻辑。当前事件/商店最小可用 UI 已落地在 `index.html + css/main.css + Game` 的接线层中。

## 已落地：事件/商店 MVP（AI-EVT-UI-01）

- 新增事件/商店面板容器：`#event-shop-panel`
- 事件展示：最近事件摘要 + 事件时间线（翻开/进行中/结算）
- 商店展示：`基础/进阶/精英` 三档位入口
- 商店行为：
  - 商店开局锁定，需先触发商人事件
  - 档位进入与同幕回访
  - 商品列表与购买按钮
  - 刷新按钮，按刷新次数递增金币消耗
  - 金币不足时给出明确反馈

## 当前职责划分

- `Game`：事件与商店 UI 的状态与交互接线（最小实现）
- `CardUI`：手牌、抽牌、牌库/弃牌堆交互（保持原职责）
- `EventBus`：HUD/手牌能量等通用事件分发

## 下一步建议拆分

- `EventPanel`：只维护事件摘要与时间线渲染
- `ShopPanel`：只维护档位、商品、购买与刷新交互
- `CurrencyBadge`：统一金币/资源显示

## 拆分原则

- 组件只处理展示和交互，不直接改游戏核心状态
- 业务状态由 `Game` 或后续 `Run/Act` 层管理，UI 通过回调或事件订阅接入
- 优先保证主循环可读性：翻格 -> 事件 -> 处理 -> 回访
