## 当前阶段结论

项目已完成 `Phase 2` 的核心收尾并正式切入 `Phase 3`：

- 扫雷核心玩法稳定可用（首击安全、泛洪展开、胜负判定）
- 卡牌系统已接入主循环（Deck/Hand/Card/UI/Effect）
- 5 张基础卡牌均可使用，且效果已接入实际游戏逻辑
- GitHub Pages 自动部署可用，页面可显示构建版本短哈希
- 交互与可维护性收口：统一提示文案、取消交互、debug 开关、冒烟清单
- 状态流与事件命名文档已整理，支持进入 Roguelike 核心开发

> 现阶段定位：**Alpha 可玩版本 + 玩法主循环重构阶段（v0.3.0-alpha）**。

### 关键设计决策（2026-03-20）

为解决“熟悉扫雷的玩家可不依赖卡牌直接通关”的核心问题，`Phase 3` 采用以下主线改造：

- 将传统地雷从“失败判定点”改造为“怪物显形点”
- 玩家揭示到地雷后进入战斗处理，而非立即失败
- 卡牌从“可选辅助”升级为“核心解题手段”
- 保留扫雷信息层（邻格数字与推理价值），并叠加战术层（卡牌战斗）

## 下一阶段计划（Phase 3，已启动）

`Phase 3` 目标是把“单局可玩”升级为“卡牌主导的可持续 Run 体验”：

- 怪物化扫雷基础规则（Mine -> Monster）
- 回合与资源循环（抽牌 / 能量 / 敌方意图 / 回合结算）
- 战斗结果与失败条件重构（揭雷不即死，失误通过资源与血量结算）
- 关卡推进与难度曲线（Level）
- 事件系统与奖励三选一（Event / Reward）
- Run 状态与基础存档（LocalStorage）
- 玩家成长系统（Perk / Meta，轻量起步）

### Phase 3 子阶段拆分（执行顺序）

1. **P3-A：玩法闭环重构（最高优先级，已完成）**
   - 完成 Mine -> Monster 的最小规则闭环
   - 打通“揭示 -> 战斗处理 -> 结算 -> 继续探索”流程
   - 验证“会扫雷但不会出牌”无法稳定通关
   - 完成状态：
     - 揭雷进入怪物遭遇（带 HP/反击），不再直接判负
     - 卡牌可处理遭遇（`scout`/`mine_detector`），并接入结算
     - 支持硬闯但需承受伤害，失败条件改为生命耗尽
     - 遭遇结算后触发回合循环（能量重置 + 手牌补充）
     - 安全格清空后若仍有未处理怪物，禁止直接胜利

2. **P3-B：Run 骨架搭建**
   - 关卡推进、事件、奖励三选一最小可用
   - Run 过程状态可保存/加载

3. **P3-C：内容与平衡扩展**
   - 扩展卡池与怪物类型
   - 打磨难度曲线与流派可玩性
   - 补充可观测数据与平衡回归清单
